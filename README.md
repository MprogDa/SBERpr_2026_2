# 1. User Story
User
- Я как пользователь системы хочу регистрироваться и входить в систему, чтобы получить доступ к своим данным и рабочим пространствам.
- Я как пользователь системы хочу восстанавливать доступ к аккаунту через email, если забыл пароль.
- Я как пользователь системы хочу создавать рабочие пространства.
- Я как пользователь системы хочу настраивать рабочие пространства (редактировать название, удалять).
- Я как пользователь системы хочу создавать задачи.
- Я как пользователь системы хочу редактировать и удалять задачи.
- Я как пользователь системы хочу назначать права участникам рабочих пространств.
- Я как пользователь системы хочу менять статусы задач, перемещая их между группами (колонками), чтобы отслеживать прогресс работы.
- Я как пользователь системы хочу фильтровать и искать задачи.
- Я как пользователь системы хочу оставлять комментарии к задачам.
- Я как пользователь системы хочу прикреплять файлы к задачам.
Admin
- Я как администратор системы хочу управлять учетными записями пользователей (в том числе менять их статус активности).
- Я как администратор системы хочу просматривать системные логи (из внешнего хранилища).
- Я как администратор системы хочу удалять рабочие пространства.
- Я как администратор системы хочу просматривать рабочие пространства.

# 2. Требования 
## Функциональные требования
### Управление пользователями и аутентификация
- Система должна позволять пользователям регистрироваться по email и паролю
- Система должна обеспечивать восстановление пароля через email (с использованием временного уникального хеша)
- Система должна обеспечивать аутентификацию пользователя
- Система должна поддерживать управление профилем пользователя (логин, пароль, email)
- Система должна поддерживать управление статусом активности пользователя (активен/заблокирован)
- Система обеспечивает разделение ролей (Админ, пользователь)
### Управление рабочими пространствами (досками)
- Система должна позволять создавать, редактировать и удалять рабочие пространства (доски)
- Система должна позволять редактировать доступ к рабочим пространствам (доскам)
- Система должна позволять назначать роли участникам
- Система должна обеспечивать приватность (доступ только владельцу)
### Управление задачами
- Внутри колонки пользователь может создать задачу
- Задачи отображаются внутри своих колонок
- Пользователь может редактировать и удалять задачи
- Возможность использования разметки Markdown в описании задач
- Пользователь может оставлять комментарии к задачам
- Пользователь может прикреплять файлы к задачам
## Нефункциональные требования
- Язык программирования: Java
- Основной фреймворк: Spring Boot
- Безопасность: Spring Security
- База данных: H2
- Сборщик проектов: Maven
- Для стилизации интерфейса используется Semantic UI
- Хранение файлов: Amazon S3 или MinIO
- Хранение логов: внешнее хранилище (отдельная реализация, не в БД)

## User (Пользователь)

| Название атрибута | Тип данных | Описание    |
| ----------------- | ---------- | ----------- |
| id                | long       | PK          |
| username          | String     |             |
| password          | String     |             |
| email             | String     |             |
| role              | String     | user, admin |
| is_active         | boolean    | Статус активности пользователя (активен/заблокирован) |
| recovery_hash     | String     | Уникальный временный хеш для восстановления пароля |
| recovery_hash_expiration | DateTime | Время истечения (жизни) хеша для восстановления пароля |

| Название атрибута | Тип данных | Описание                                        |
